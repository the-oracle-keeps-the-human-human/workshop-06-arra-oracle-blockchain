## 2.1 การเตรียมบัญชีและค่าแก๊ส (L1 Pool Wallet)

เมื่อต้องการเชื่อมโยง Layer 2 เข้ากับ Layer 1
กระบวนการยิงสัญญาบน Sepolia Testnet ก็จะเกิดขึ้น
โดยกระเป๋ากลางสำหรับจ่ายค่าแก๊ส (L1 Pool Wallet)
จะทำหน้าที่จ่ายค่าธรรมเนียมทั้งหมดในตอนเริ่มต้นนะ

ที่อยู่กระเป๋าเงินกองกลางนี้คือ:
`0x644Da211BB604B58666b8a9a2419E4F3F2aceC0A`

กุญแจสำคัญที่ใช้สั่งการยิงธุรกรรม
มีรหัสลับหรือ Private Key ดังนี้:
`0xf21bcb0b78c1731e34416b9b6da9e7de033cbe96a4439b6cd4c1324a2f205204`

พอระบบมีงบประมาณเริ่มต้น 2.75 Sepolia ETH
การจัดสรรค่าแก๊สไปสู่บัญชีอื่นก็เริ่มขึ้น
แต่ระบบไม่อาจปล่อยให้กระเป๋าใบเดียวรับภาระทั้งหมด
เนื่องจากมีความเสี่ยงในการส่งธุรกรรมซ้อนกัน

เมื่อกระจายบัญชีตามหน้าที่
ก็ช่วยให้ระบบมีเสถียรภาพขึ้น
โดยโอนแยกไปให้บัญชีละ 0.05 Sepolia ETH
ซึ่งประกอบด้วยผู้มีหน้าที่สามส่วนสำคัญ:

1. **Proposer Wallet** รับหน้าที่ยื่นข้อพิพาทและยืนยันรัฐ L2
2. **Batcher Wallet** ทำหน้าที่อัปโหลดบล็อกดิบไปลง L1
3. **Sequencer Wallet** คอยทำหน้าที่จัดลำดับบล็อกในระบบ

## 2.2 op-deployer apply: ขั้นตอนการยิงธุรกรรมจริง 17+ ธุรกรรม

เมื่อเตรียมทรัพยากรบน Sepolia พร้อมแล้ว
การสั่งติดตั้งสัญญาก็จะเริ่มทำงานทันที
โดยการรันคำสั่ง op-deployer apply
ผ่านกุญแจกองกลางเมื่อเวลา 07:33 UTC

การดำเนินการนี้ทำโดย Nova
ซึ่งสัญญาทั้งหมดจะถูกเขียนลงไปในบล็อก
มีการยืนยันธุรกรรมมากกว่า 17 ธุรกรรมทีเดียว

ตัวอย่างคำสั่งสำหรับใช้งาน:
```bash
rtk op-deployer apply \
  --config intent.toml \
  --l1-rpc https://sepolia.infura.io/v3/KEY
```

พอธุรกรรมได้รับการประมวลผลเสร็จสิ้น
ก็สามารถสรุปสถิติสำคัญออกมาได้ครับ:
- **Transaction Hash**
  `0x3936eecf2fe2fb948b56c721418e4e0b0afc887f72511d21fbcb2a6342b43979`
- **Gas Used** สิ้นเปลืองแก๊สไป 10,656,681 gas
- **Effective Price** อัตราแก๊สอยู่ที่ 2.148 Gwei
- **Total Cost** รวมค่าใช้จ่ายทั้งหมด ~0.15 ETH
- **Start Block** เริ่มลงทะเบียนที่ L1 บล็อก 11092765

ธุรกรรมชุดนี้ได้รับการยืนยันครบถ้วน
และพร้อมที่จะเปิดใช้ฟังก์ชันของ L2 แล้วครับ

## 2.3 ข้อพิจารณาของ Contracts สำคัญ: OptimismPortal, SystemConfig, DisputeGameFactory

เมื่อการ deploy บรรลุผลสำเร็จ
สัญญาฝั่ง L1 ก็จะทำงานประสานกัน
โดยมีสัญญาอัจฉริยะสำคัญสามตัวที่ควรระวังครับ

ตัวแรกคือ **OptimismPortalProxy**
อยู่ที่ตำแหน่ง:
`0xcDAd5Bf85455DA5b3EfF8fFef1f8Ba5cc49d7011`
ทำหน้าที่คอยประสานงาน deposit และ withdraw
หากมีผู้ใช้งานส่งสินทรัพย์ข้ามขอบข่าย
สัญญานี้ก็จะเป็นตัวจัดการดูแลให้ทันทีเลย

ตัวถัดมาคือ **SystemConfigProxy**
อยู่ที่ตำแหน่ง:
`0xd4645B54EC1192b11d348Ffcb1008d87A4C64C59`
เป็นพื้นที่เก็บค่าคอนฟิกสำคัญของระบบ L2
เมื่อมีการระบุ Batcher Address หรือกำหนดอัตรา Gas Limit
ระบบก็จะดึงค่าตั้งต้นมาจากจุดนี้แหละ

สัญญาที่สามคือ **DisputeGameFactoryProxy**
อยู่ที่ตำแหน่ง:
`0x3E5c2BfcA48aD45826129b4e66190B9b5F58E3bd`
เป็นตัวควบคุมระบบการพิสูจน์ความถูกต้อง (Fault Proof)
เมื่อเกิดความขัดแย้งเกี่ยวกับบล็อกที่ผลิต
สัญญานี้ก็จะเป็นผู้ปั๊มตัวเกมเพื่อหาข้อยุติครับ

แต่ในการติดตั้งจริงยังมีสัญญาบริวารอื่นอีก
โดย Orz ได้ทำการวิเคราะห์

## 2.4 หลุมพรางของ intent.toml: ปัญหารูปแบบ Fee Vaults และ overrides

แต่ปัญหาที่พบได้บ่อยมากในการ deploy
มักจะมาจากไฟล์คอนฟิกหลัก intent.toml
ซึ่งจุดที่ควรระมัดระวังมีอยู่สองส่วนด้วยกันนะ

หลุมพรางที่หนึ่งคือ รูปแบบของ Fee Vaults
เมื่อใช้ค่าเริ่มต้นในระบบ
สัญญากลุ่มเก็บค่าธรรมเนียมจะกำหนดค่าเป็นศูนย์:

```toml
# ❌ การเซ็ตอัปแบบเดิมที่ทำให้ระงับการทำงาน
sequencer_fee_vault = "0x000...0"
l1_fee_vault = "0x000...0"
base_fee_vault = "0x000...0"
```

พอสั่งใช้งานผ่าน op-deployer
ตัวระบบก็จะเกิดข้อผิดพลาดและยุติคำสั่งทันที
เมื่อต้องการแก้ไขให้รันผ่านได้
ก็ต้องระบุที่อยู่กระเป๋าจริงลงไปแทนเลขศูนย์นะครับ

หลุมพรางที่สองคือ บล็อกก๊สพิเศษ (customGasToken)
หากทิ้งบล็อกข้อมูลนี้ไว้ในลักษณะว่างเปล่า:

```toml
# ❌ การทิ้งบล็อกก๊สพิเศษไว้ให้สับสน
[chains.customGasToken]
```

เมื่อปล่อยให้มีข้อมูลนี้ค้างอยู่
ระบบก็อาจจะชนกับ standard-overrides
จนเกิดข้อความล้มเหลวแบบ incompatible
พอทำการลบส่วนนี้ออกไปทั้งกลุ่ม
ปัญหาการตั้งค่าทั้งหมดก็คลี่คลายไปได้ทีเดียว
